Documentation Class ZipTextSearcher:

  1. Directory Definition: $zipDirectory = 'zips/'; defines the directory where your ZIP files should be placed. Make sure to create this directory in the same location as your PHP script.

  2. Get ZIP Files: glob($zipDirectory . '*.zip') finds all files ending with .zip in the /zips directory and stores them in the $zipFiles array.

  3. Form Structure:

    • A basic HTML form with the method="post" and action="" (submits to the same script).
    • Search String Input: An <input type="text"> with the name search_text. The value attribute is populated with the previously entered search string (if any) using htmlspecialchars() for security.
    • ZIP File Select Dropdown: A <select> element with the name zip_file.
      • The first <option> is a default "Select a ZIP File" option.
      • A PHP loop iterates through the $zipFiles array and creates an <option> for each ZIP file. The value of each option is the full path to the ZIP file, and the displayed text is just the filename (using basename()).
    • Search Button: A <button type="submit"> to submit the form.

  4. Form Submission Handling (if ($_SERVER['REQUEST_METHOD'] === 'POST')):

    • Checks if the form has been submitted using the POST method.
    • Get Search String: Retrieves the value of the search_text input using $_POST['search_text'] and trims any leading or trailing spaces using trim().
    • Get Selected ZIP File: Retrieves the value of the zip_file select dropdown using $_POST['zip_file'].
    • Validation: Checks if both the search string and a ZIP file have been selected and if the selected file is actually in the $zipFiles array.
    • Instantiation and Search: If the validation passes:
      • A new instance of the ZipTextSearcher class is created with the path to the selected ZIP file.
      • The searchInZip() method is called with the trimmed search string.
    • Error Handling: A try...catch block is used to handle the InvalidArgumentException that might be thrown by the ZipTextSearcher constructor if the ZIP file is invalid.
    • Empty Input Error: If the search string or ZIP file is not selected, an error message is set.

  5. Displaying Results:

    • After processing the form, the script checks if there were any errors and displays them in a red <p> tag.
    • If the search was performed ($searchResults is not empty and the form was submitted correctly), a "Search Results" section is displayed.
    • It iterates through the $searchResults array and shows the filename and the lines where the search string was found.
    • If no matches were found, a "No matches found" message is displayed.

  6. Displaying the Report:

    • Finally, if a $zipSearcher object was successfully created (meaning a valid ZIP file was processed), the generateReport() method is called to display the search report.

To Use This:

  1. Save the code: Save the entire code (including the class definition and the HTML/PHP section) as a .php file (e.g., zip_search.php).
  2. Create the zips directory: In the same directory where you saved the zip_search.php file, create a new directory named zips.
  3. Place your ZIP files: Put the .zip files you want to search within into the /zips directory.
  4. Access in your browser: Open the zip_search.php file in your web browser. You should see the form with the dropdown list of your ZIP files.
  5. Enter search string and select file: Enter the text you want to find in the "Search String" field, choose a ZIP file from the dropdown, and click the "Search" button.
  6. View results and report: The search results (if any) and the search report will be displayed below the form.

Remember to ensure that your web server has the necessary permissions to read files in the /zips directory.
